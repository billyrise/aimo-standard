# Conformidad

Esta sección define cómo las organizaciones reclaman cumplimiento con el AIMO Standard y qué evidencia se espera.

## Propósito

- **Qué puede reclamar**: Alineación con la estructura de evidencia y requisitos de trazabilidad del AIMO Standard.
- **Qué no puede reclamar**: Cumplimiento legal, certificación o aprobación regulatoria. Consulte [Límite de Responsabilidad](../governance/responsibility-boundary/).

## Evidencia Esperada

Para demostrar conformidad, las organizaciones deben proporcionar:

1. **Paquete de Evidencia**: Registros EV estructurados según [Artefactos](../artifacts/) y [Requisitos Mínimos de Evidencia](../artifacts/minimum-evidence/).
2. **Resultados del Validador**: Salida aprobada del validador — consulte [Validador](../validator/).
3. **Referencia de Versión**: Versión explícita del AIMO Standard — consulte [Versiones](../standard/versions/).

## No sobre-reclamar

!!! warning "Importante"
    La conformidad con el AIMO Standard demuestra **preparación de evidencia y trazabilidad**, no cumplimiento legal ni certificación. Consulte [Límite de Responsabilidad](../governance/responsibility-boundary/).
