# 授權

本頁記錄 AIMO 標準材料的授權條款。

## 授權類型

AIMO 標準根據 **Apache License, Version 2.0** 授權。

- SPDX 識別碼：`Apache-2.0`
- 完整文本：[Apache License 2.0](https://www.apache.org/licenses/LICENSE-2.0)

## 範圍

### 涵蓋內容

| 材料 | 授權 | 位置 |
| -------- | ------- | -------- |
| 文件 | Apache-2.0 | `docs/` |
| JSON 結構描述 | Apache-2.0 | `schemas/` |
| 範本 | Apache-2.0 | `templates/` |
| 範例 | Apache-2.0 | `examples/` |
| 驗證器 | Apache-2.0 | `validator/` |
| 工具 | Apache-2.0 | `tooling/` |
| PDF 發布 | Apache-2.0 | GitHub Releases |
| 人工產物 ZIP | Apache-2.0 | GitHub Releases |

### 您可以做的事

根據 Apache-2.0，您可以：

- 將材料用於任何目的（商業或非商業）
- 修改並建立衍生作品
- 發布原始或修改版本
- 包含在專有產品中

### 要求

使用 AIMO 標準材料時：

1. 在發布中包含授權聲明
2. 提供署名（請參閱 [NOTICE.txt](https://github.com/billyrise/aimo-standard/blob/main/NOTICE.txt)）
3. 說明重大修改（如有）
4. 未經許可不得使用商標（請參閱[商標](../trademarks/)）

## NOTICE 檔案

[NOTICE.txt](https://github.com/billyrise/aimo-standard/blob/main/NOTICE.txt) 檔案包含必要的署名資訊。

## 第三方元件

AIMO 標準可能包含或參照第三方元件。每個元件保留其原始授權。

## 常見問題

**問：我可以在商業產品中使用 AIMO 標準嗎？**
答：可以，Apache-2.0 允許商業使用。

**問：我需要開源我的實作嗎？**
答：不需要，Apache-2.0 不要求衍生作品開源。

**問：我可以修改結構描述嗎？**
答：可以，但您必須註明修改，且不能以修改後的結構描述聲明符合性。
